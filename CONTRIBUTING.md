# Contributing

## Adding your project

1. Create `projects/<your-project-name>/`
2. Add a `README.md` with:
   - One-line description
   - Team members
   - Sponsor tools used (Nimble / ClickHouse / Senso / payment rails / other)
   - How to run locally
   - Demo video link (once you have it)
3. Push to `main` directly or open a PR — don't block on review during the sprint

## Folder layout

```
projects/
  your-project-name/
    README.md
    <code, configs, whatever you need>
```

Pick whatever stack ships fastest. Mix languages across teams is fine.

## Submission checklist

Before 4:30 PM on May 23:

- [ ] Code pushed to `main` under `projects/<your-name>/`
- [ ] Project README filled in (team, tools used, run instructions)
- [ ] 3-min demo recorded and linked from your README
- [ ] Devpost submission completed (link your GitHub folder)
- [ ] At least 2 sponsor tools integrated (3+ for top tool-use score)

## Ground rules

- **Ship something.** A rough working demo beats a polished plan.
- **No previously built projects.** Devpost rule — start fresh today.
- **No secrets in commits.** Use `.env.local`, gitignored.
- **Real sources, real work.** Judging rewards autonomy and grounding, not vibes.
- **Be kind in PR reviews.** Hackathon, not prod.

## Questions

Open an issue, ask in the team channel, or email team@creatorscorner.co.
